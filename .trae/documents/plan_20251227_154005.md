## 修复计划：玻璃态模糊拖动条视觉反馈问题

### 问题分析

1. **无 CSS 过渡效果**：CanvasBackground 中的 `backdrop-filter` 没有 `transition` 属性，导致拖动时视觉变化不连贯

2. **状态更新延迟**：Zustand store 每次更新都递增 `version` 触发全量重渲染，可能有轻微延迟

3. **浏览器渲染优化**：某些浏览器对 `backdrop-filter` 的更新有节流处理

### 修复方案

**1. 修改 `CanvasBackground.tsx`**
- 为玻璃态模糊层添加 CSS `transition: backdrop-filter 150ms ease-out`
- 添加 `will-change: backdrop-filter` 提示浏览器优化
- 添加 `transform: translateZ(0)` 触发 GPU 加速

**2. 修改 `SliderRow.tsx`（可选增强）**
- 添加拖动时的即时视觉反馈（滑块颜色变化）
- 优化触摸/鼠标事件响应

**预期效果**：拖动玻璃模糊滑块时，视觉效果会实时、平滑地变化，消除延迟感。
