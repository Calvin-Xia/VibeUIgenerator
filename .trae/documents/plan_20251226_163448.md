## Noise 选项功能异常问题修复方案

### 问题分析

通过代码检查发现 Noise 功能的问题：

1. **动画触发条件错误**（CanvasBackground 第 25 行）：
   ```typescript
   className={cn(
     'absolute inset-0 z-0 overflow-hidden rounded-2xl',
     showNoise && 'animate-noise'  // 使用的是 UI 状态，不是 effects.noise.enabled
   )}
   ```
   - `showNoise` 是 UI 的开关状态，默认为 `false`
   - `animate-noise` 动画绑定到错误的条件

2. **噪点纹理不够明显**（CanvasBackground 第 35-40 行）：
   - `opacity-30` 类固定设置透明度为 0.3
   - 没有使用动态的 `effects.noise.intensity` 值
   - 噪点 SVG 的 baseFrequency 可能不够明显

### 修复方案

修改 `CanvasBackground.tsx`：

1. **修复动画触发条件**：
   - 将 `showNoise && 'animate-noise'` 改为 `effects.noise.enabled && 'animate-noise'`

2. **增强噪点纹理显示**：
   - 动态应用 `effects.noise.intensity` 到样式
   - 适当提高噪点的对比度和可见度
   - 移除固定的 `opacity-30` 类，使用动态内联样式

### 修改文件
- `components/preview/CanvasBackground.tsx`

### 预期效果
- 勾选 Noise 的 Enabled 选项后，噪点纹理立即显示
- 噪点呈现动态抖动动画效果
- 噪点纹理清晰可见
